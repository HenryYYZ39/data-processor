## 当前项目

### data-processor（数据合并处理工具）
- **功能**：4个上传框 → 合并Excel（新增机床工况数据）
- **版本**：v9
- **状态**：已完成机床数据处理功能

**关键代码位置**：
- 版本号：第6行
- 数据循环：第767行 (`i = 2` 跳过第二行)

**核心功能**：
1. 工况识别：日期→时间→内容（分行显示）
2. 温度列：自动识别Untitled列（5-40°C）
3. 误差计算：与标定值做差值归一化
4. **机床工况**：1秒重采样、秒数转时间、匹配插入Sheet2

## 经验

### 技术
- SheetJS 处理Excel，Chart.js 可画图表
- **时间解析**：分别匹配阿拉伯数字时间（`^\d{1,2}:\d{2}`）和中文时间，用 `^` 锚定行首；避免全局替换预处理
- 正则表达式识别日期/时间格式
- 版本号加在title标签，方便强制刷新缓存
- Git 回退：`git reset --hard <commit>` + `git push --force`
- **Excel时间格式**：直接存储格式化字符串，避免Date对象默认格式问题
- **时间匹配**：两端格式必须完全一致（包括毫秒、AM/PM）

### 协作
- GitHub token会过期，需要定期更新
- 网络不稳定时push会失败，需要重试
- 截图反馈比文字描述更直观
- 功能改动前先确认需求，避免来回撤销

## 教训

- 变量名要检查：`idx` → `i` 的bug
- 合并冲突时先pull再push
- 复杂功能分步骤实现，不要一次改太多
- **需求确认要精确**：时间格式必须问清（是否带毫秒、12/24小时制）
- **Excel显示vs存储**：单元格"显示格式"和"实际存储值"是两个概念
