## 经验

### 技术
- SheetJS 处理Excel，Chart.js 可画图表
- **时间解析**：分别匹配阿拉伯数字时间（`^\d{1,2}:\d{2}`）和中文时间，用 `^` 锚定行首；避免全局替换预处理
- 正则表达式识别日期/时间格式
- 版本号加在title标签，方便强制刷新缓存
- Git 回退：`git reset --hard <commit>` + `git push --force`
- **Excel时间格式**：直接存储格式化字符串，避免Date对象默认格式问题
- **时间匹配**：两端格式必须完全一致（包括毫秒、AM/PM）
- **Chart.js大数据优化**：超过5000点时均匀采样，禁用动画和贝塞尔曲线，渲染速度提升10-20倍
- **分批异步处理**：使用 `await new Promise(r => setTimeout(r, 0))` 让出主线程，避免大数据处理卡死
- **Map索引加速**：用 `Map<时间字符串, 数据数组>` 实现O(1)时间匹配

### 协作
- GitHub token会过期，需要定期更新
- 网络不稳定时push会失败，需要重试
- 截图反馈比文字描述更直观
- 功能改动前先确认需求，避免来回撤销

## 教训

- 变量名要检查：`idx` → `i` 的bug
- 合并冲突时先pull再push
- 复杂功能分步骤实现，不要一次改太多
- **需求确认要精确**：时间格式必须问清（是否带毫秒、12/24小时制）
- **Excel显示vs存储**：单元格"显示格式"和"实际存储值"是两个概念
- **大数据处理必须分批+异步**：否则浏览器会卡死
- **图表渲染必须采样**：Canvas绘制大量路径性能瓶颈明显，超过5000点必须采样
