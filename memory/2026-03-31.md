## 经验

### 前端性能优化
- **Chart.js大数据量优化**：超过5000点时均匀采样，禁用动画和贝塞尔曲线，渲染速度提升10-20倍
- **分批异步处理**：大数据处理使用 `await new Promise(r => setTimeout(r, 0))` 让出主线程，避免卡死

### 数据处理
- **时间格式统一**：确保传感器时间和工况时间格式一致（yyyy/m/d h:mm:ss），否则匹配失败
- **动态列数支持**：使用 `slice(1)` 和动态长度计算，支持任意列数的工况数据
- **Map索引加速**：用 `Map<时间字符串, 数据数组>` 实现O(1)时间匹配

### 代码结构
- **步骤面板管理**：通过 `classList.remove('disabled')` 控制面板可点击状态
- **版本号强制刷新**：URL加 `?v=版本号` 或时间戳，避免浏览器缓存

### Git工作流
- 网络不稳定时push会失败，需要重试
- 冲突时使用 `git pull --rebase` 解决

## 教训
- 时间格式必须严格一致，包括是否补零、12/24小时制、AM/PM标记
- 大数据处理必须分批+异步，否则浏览器会卡死
- 图表渲染必须采样，Canvas绘制大量路径性能瓶颈明显
